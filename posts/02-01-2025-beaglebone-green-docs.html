Getting Started with BeagleBone Green

Published on January 2, 2025

I'm sure you've heard of Raspberry Pi numerous times. It's a real neat board but have you ever heard of an alternative? Let me introduce you to BeagleBone Green (BBG)! Both boards represent something called single-board computer (SBC). Essentially it's like a micro computer you can buy for a few dozens bucks

What is an SBC? ðŸ¤”

SBC integrates all components of a traditional computer onto one board, including the processor, memory, and I/O interfaces. The BeagleBone Green, for that matter, features:

  • AM335x 1GHz ARMR Cortex-A8
  • 512MB DDR3 RAM
  • 4GB eMMC storage
  • 92 pins

There are tons of project ideas available on the internet with people doing all sorts of crazy IoT stuff with Raspberry or BeagleBone ðŸ˜…. The Arm32 architecture gives them a huge advantage over "regular" Arduino projects - you can install Linux OS, connect to the internet or even use with AI.

Calculating Pin Numbers on the BeagleBone Green

I must admit that as for today BBG is poorly documented and I strggled quite a bit to get basic things done. That's why I decided to share some essentials I figured while learning so that you don't have to hassle your way through the whole thing xd. If you're switching from Arduino or STM32 you're probably gonna get a little confused with the pins location. Only the pins on the very corners on the board and the pin blocks ("headers") are described.

But are the numbers actually correct? When you look at the graphics from the wiki, you'll notice discrepancy between GPIO numbers and pin numbers. So who's right? Well, in reality the 96 available pins on the board DO NOT represent the internal CPU's architecture in 100%. Only subset of pins is exposed. The AM335x CPU is split into 4 "GPIO banks" 32 pins each. So in theory you have 128 GPIO pin although not all of them are at your disposal. Now, you'll often find yourself translating the pins between two naming conventions: with and without bank number provided explicitly:

  • GPIO1_17 (1 * 32 + 17) == GPIO 49
  • GPIO0_4 (0 * 32 + 4) == GPIO 4
  • GPIO3_11 (3 * 32 + 11) == GPIO 107

Now, as you can see on the graphics mentioned earlier it has nothing to do with the pin numbers on the board (1-46 on each side of the board). For example, GPIO1_17 refers to pin 17 in GPIO bank 1, which corresponds to GPIO 49 in the system, and can be found on pin 23 in header P9. So using the wiki as our documentation we have:

  • GPIO1_17 (1 * 32 + 17) == GPIO 49 == pin 23 in P9
  • GPIO0_4 (0 * 32 + 4) == GPIO 4 == pin 18 in P9
  • GPIO3_11 (2 * 32 + 11) == GPIO 75 == pin 42 in P8

But this is not the last conversion you will encounter! The pins have some constants pre-defined in order to tweak configuration of your board (like set a pin to be PWM instead of GPIO etc) and are available here: am33xx.h. Notice that the names don't quite correspond to anything we've seen in the pins for example what "AM335X_PIN_GPMC_A1" means? On the wiki there are some reference files at the bottom, this one for example shows the cryptic names if you Ctrl+F for "GPMC_A1" you'll find this lengthy line "GPMC_A1/GMII2_RXDV/RGMII2_RCTL/MMC2_DAT0/GPMC_A17/PR1_MII1_TXD3/EHRPWM1_SYNCI_O/GPIO1_17" meaning that the pin is GPIO1_17 -> GPIO 49 -> pin 23 in P9. When you will find yourself modifying .dts files you will need the constants from the .h file. To complete the example:

  • GPIO1_17 (1 * 32 + 17) == GPIO 49 == pin 23 in P9 == AM335X_PIN_GPMC_A1 (or 0x844)
  • GPIO0_4 (0 * 32 + 4) == GPIO 4 == pin 18 in P9 == AM335X_PIN_UART0_RXD (or 0x970)
  • GPIO2_11 (2 * 32 + 11) == GPIO 75 == pin 42 in P8 == AM335X_PIN_LCD_DATA5 (or 0x8b4)
